# Описание проекта: Make-a-Lisp (1С:Предприятие)

Данный репозиторий представляет собой заготовку для реализации проекта [Make-a-Lisp](https://github.com/kanaka/mal) (создание интерпретатора языка Lisp шаг за шагом) на встроенном языке программирования 1С:Предприятие.

## Текущая структура и архитектура

Исходные коды представлены в виде выгрузки конфигурации 1С в формат XML.
Основная точка входа и пользовательский интерфейс расположены в обработке `MakeALisp` (`src/DataProcessors/MakeALisp`).

### Пользовательский интерфейс (UI / REPL)
- **Путь:** `src/DataProcessors/MakeALisp/Forms/Form`
- **Модуль формы:** `src/DataProcessors/MakeALisp/Forms/Form/Ext/Form/Module.bsl`
- **Функционал:**
  - Интерактивная оболочка для ввода и выполнения кода Lisp.
  - Кнопки для загрузки (`Load`) и сохранения (`Save`) скриптов в текстовые файлы `.txt`.
  - Возможность включения/выключения отображения журнала отладки (`DebugLog`).
  - Выбор текущего шага реализации (от `Step 0` до `Step A`). При выборе шага автоматически подгружается соответствующий тестовый код из макетов (Templates) обработки (например, `Test0`, `Test1` и т.д.).

### Состояние реализации интерпретатора
- В процедуре `CallLispInterpreter` (в модуле формы) заложена логика маршрутизации вызовов интерпретатора в зависимости от выбранного шага:
  ```bsl
  // Output = Step0.MaL_Step_0(Input, DebugLog);
  // ...
  // Output = StepA.MaL_Step_A(Input, DebugLog);
  ```
- **На данный момент:** Шаги `Step 0` и `Step 1` успешно реализованы. Модули `Step0` и `Step1` загружены, вызовы в UI раскомментированы и тесты проходят успешно. Остальные шаги ожидают реализации.

## План дальнейшей работы
В рамках этого проекта необходимо будет последовательно реализовывать логику интерпретатора Lisp на языке 1С в соответствии с шагами:
- [x] **Step 0:** Настройка REPL (Read-Eval-Print Loop) - чтение строки, возврат той же строки.
- [x] **Step 1:** Read and Print - реализация парсера (разбор строки в AST) и принтера (вывод AST в строку).
- [ ] **Step 2:** Eval - реализация базового вычисления выражений.
- **Step 3:** Environments - реализация окружения для хранения переменных и базовых функций (let, def!).
- **Step 4:** If, Fn, Do - условные операторы, создание пользовательских функций.
- И так далее вплоть до **Step A** (полноценный интерпретатор с поддержкой макросов, обработки ошибок, хвостовой рекурсии и взаимодействия с файловой системой).
