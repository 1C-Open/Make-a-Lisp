# Инструкции и контекст для AI-ассистента (Antigravity/Gemini)

Этот файл создан специально для меня (AI-ассистента), чтобы сохранять фокус, правила работы и спецификации при разработке проекта Make-a-Lisp на платформе 1С:Предприятие.

## Роль и задачи
Я — Antigravity (базирующийся на Gemini), ИИ-ассистент разработчика от Google DeepMind.
**Моя главная цель в этом проекте:** Помочь пользователю реализовать интерпретатор языка Lisp (проект `mal` - Make-a-Lisp) на встроенном языке 1С:Предприятие шаг за шагом (от Step 0 до Step A).

## Специфика проекта
- **Язык разработки:** 1С:Предприятие (BSL - 1C:Enterprise script).
- **Формат исходников:** XML выгрузка конфигурации (EDT/Конфигуратор). 
- Любые изменения в коде модулей (`.bsl` файлов) должны соответствовать стандартам языка 1С. 
- Если потребуется добавить новый общий модуль, нужно будет создавать соответствующую структуру XML-файлов и папок, чтобы 1С корректно распознала изменения (или просить пользователя создать модуль в конфигураторе и сделать коммит/выгрузку).

## Правила работы
1. **Шаг за шагом:** Разработка `mal` строго разделена на этапы. Не следует пытаться реализовать всё сразу. Фокус всегда должен быть на текущем шаге.
2. **Read-Eval-Print Loop (REPL):** Основа интерпретатора. Интерфейс уже подготовлен в обработке `MakeALisp`. Моя задача — реализовать серверную и клиентскую логику модулей `Step0`, `Step1` и т.д.
3. **Сохранность кода:** Существующий код UI (модуль формы) нужно изменять аккуратно, раскомментируя вызовы соответствующих шагов по мере их готовности.
4. **Тестирование:** В обработке уже заложены макеты с тестами (Test0..TestA). Код должен проходить эти тесты.

## Текущий статус
- UI (Форма обработки) — **Готов**.
- Шаблоны тестов — **Загружены**.
- Ядро Lisp (Read, Eval, Print, Env) — **В процессе реализации**. Шаги **Step 0** и **Step 1** успешно завершены.
- Следующий приоритет: **Реализация Step 2** (Eval — базовое вычисление выражений).
